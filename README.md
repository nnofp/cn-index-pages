# cn-index-pages

本仓库用于归档和发布多个独立 HTML 页面。这些页面以静态文件的形式存放，不针对任何特定域名或具体网站，仅作为通用页面集合使用。

## 项目简介

本项目旨在提供一个结构清晰的静态 HTML 页面仓库。收录的页面均为独立文件，不依赖外部数据库或后端服务，可直接通过浏览器访问。适合作为页面存档、示例展示或轻量级信息分发用途。

## 目录说明

```
cn-index-pages/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的静态资源（如图片、样式表、脚本）
├── index.html      # 仓库根目录入口页面
└── README.md       # 本文件
```

- **pages/**：按类别或功能划分的子目录，每个 HTML 文件可独立运行。
- **assets/**：存放页面共同使用的资源文件，避免重复引用。
- **index.html**：仓库默认首页，提供页面列表或导航链接。

## 页面归档说明

所有页面均以 HTML 格式保存，遵循 W3C 标准。归档时注意以下原则：

- 每个页面应自包含，外部依赖（如 CSS、JS）尽量放在 assets/ 目录下。
- 页面文件名使用英文小写字母、数字和连字符，避免特殊字符。
- 页面内容应保持简洁，不包含敏感信息或违规内容。

## 维护说明

本仓库欢迎提交改进或新增页面。维护时请遵循以下步骤：

1. Fork 本仓库并创建新分支。
2. 在 pages/ 下添加或修改 HTML 文件。
3. 确保页面在主流浏览器中可正常显示。
4. 提交 Pull Request，并简要说明变更内容。
